Writing Your First Django App

45 min read Updated Jan 2024 Beginner Django 4.2
Learning Objective: By the end of this tutorial, you'll have built a simple poll application with a public voting interface and admin backend.

Introduction

Welcome to your first Django application! In this tutorial, we'll build a simple poll application that demonstrates Django's core concepts and features.

The application will have two parts:

  • A public site that lets people view polls and vote in them
  • An admin site that lets you add, change, and delete polls
Public Voting

Users can view and vote on polls

Admin Interface

Manage polls through Django admin

Database Models

Store polls and votes in database

Prerequisites

Before starting, make sure you have:

  • Python 3.8 or higher installed
  • Basic understanding of Python syntax
  • Familiarity with command line/terminal
Note: If you haven't installed Django yet, check out our Installation Guide first.

Project Setup

Let's start by creating a new Django project and application.

1 Create a Project

Open your terminal and run the following command to create a new Django project:

django-admin startproject mysite

This will create a mysite directory with the following structure:

mysite/
    manage.py
    mysite/
        __init__.py
        settings.py
        urls.py
        asgi.py
        wsgi.py

2 Navigate to Project Directory

cd mysite

3 Create the Polls App

Now let's create our polls application:

python manage.py startapp polls

This creates a polls directory with the following structure:

polls/
    __init__.py
    admin.py
    apps.py
    migrations/
        __init__.py
    models.py
    tests.py
    views.py

4 Configure the Project

Open mysite/settings.py and add 'polls' to the INSTALLED_APPS list:

INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    # Our app
    'polls',
]

Creating Models

Models define the structure of your database. Let's create models for our poll application.

1 Define Models

Open polls/models.py and add the following code:

from django.db import models

class Question(models.Model):
    question_text = models.CharField(max_length=200)
    pub_date = models.DateTimeField('date published')
    
    def __str__(self):
        return self.question_text

class Choice(models.Model):
    question = models.ForeignKey(Question, on_delete=models.CASCADE)
    choice_text = models.CharField(max_length=200)
    votes = models.IntegerField(default=0)
    
    def __str__(self):
        return self.choice_text

Here we've created two models:

  • Question: Has a question text and publication date
  • Choice: Has the choice text, vote count, and is linked to a Question via ForeignKey

2 Create Migrations

Migrations are Django's way of propagating changes you make to your models into the database schema.

python manage.py makemigrations polls

You should see output similar to:

Migrations for 'polls':
  polls/migrations/0001_initial.py
    - Create model Question
    - Create model Choice

3 Apply Migrations

Now apply the migrations to create the database tables:

python manage.py migrate

Setting Up the Admin Interface

Django provides a powerful admin interface out of the box. Let's set it up for our models.

1 Create Superuser

First, create a user who can login to the admin site:

python manage.py createsuperuser

Follow the prompts to enter your username, email, and password.

2 Register Models

Open polls/admin.py and register your models:

from django.contrib import admin
from .models import Question, Choice

admin.site.register(Question)
admin.site.register(Choice)

3 Start Development Server

Start the development server and explore the admin interface:

python manage.py runserver

Open your browser and go to http://127.0.0.1:8000/admin/. Log in with the superuser account you created.

Creating Views

Views are Python functions that take web requests and return web responses. Let's create some basic views.

1 Create Your First View

Open polls/views.py and add the following view:

from django.http import HttpResponse

def index(request):
    return HttpResponse("Hello, world. You're at the polls index.")

2 Map URLs to Views

Create a urls.py file in the polls directory and add the following:

from django.urls import path
from . import views

urlpatterns = [
    path('', views.index, name='index'),
]

Now include the polls URLs in the project's mysite/urls.py:

from django.contrib import admin
from django.urls import include, path

urlpatterns = [
    path('polls/', include('polls.urls')),
    path('admin/', admin.site.urls),
]

3 Test Your View

Visit http://127.0.0.1:8000/polls/ in your browser. You should see the text "Hello, world. You're at the polls index."

Creating Database-Driven Views

Let's create views that interact with our database models.

1 Update the Index View

Modify polls/views.py to display the latest questions:

from django.http import HttpResponse
from .models import Question

def index(request):
    latest_question_list = Question.objects.order_by('-pub_date')[:5]
    output = ', '.join([q.question_text for q in latest_question_list])
    return HttpResponse(output)

2 Add More Views

Let's add views for detail pages and results:

def detail(request, question_id):
    return HttpResponse("You're looking at question %s." % question_id)

def results(request, question_id):
    response = "You're looking at the results of question %s."
    return HttpResponse(response % question_id)

def vote(request, question_id):
    return HttpResponse("You're voting on question %s." % question_id)

3 Update URLs

Add these new views to polls/urls.py:

from django.urls import path
from . import views

urlpatterns = [
    # ex: /polls/
    path('', views.index, name='index'),
    # ex: /polls/5/
    path('<int:question_id>/', views.detail, name='detail'),
    # ex: /polls/5/results/
    path('<int:question_id>/results/', views.results, name='results'),
    # ex: /polls/5/vote/
    path('<int:question_id>/vote/', views.vote, name='vote'),
]

Using Templates

Templates allow you to separate your Python code from your HTML. Let's create some templates for our views.

1 Create Templates Directory

In your polls directory, create a directory called templates, and within that, create another directory called polls:

polls/
    templates/
        polls/

2 Create Index Template

Create polls/templates/polls/index.html with the following content:

<!DOCTYPE html>
<html>
<head>
    <title>Polls</title>
</head>
<body>
    <h1>Latest Poll Questions</h1>
    {% if latest_question_list %}
        <ul>
        {% for question in latest_question_list %}
            <li><a href="{% url 'detail' question.id %}">{{ question.question_text }}</a></li>
        {% endfor %}
        </ul>
    {% else %}
        <p>No polls are available.</p>
    {% endif %}
</body>
</html>

3 Update Index View

Update the index view to use the template:

from django.shortcuts import render
from .models import Question

def index(request):
    latest_question_list = Question.objects.order_by('-pub_date')[:5]
    context = {'latest_question_list': latest_question_list}
    return render(request, 'polls/index.html', context)

Adding Forms

Let's create a form that allows users to vote on polls.

1 Create Detail Template

Create polls/templates/polls/detail.html:

<!DOCTYPE html>
<html>
<head>
    <title>{{ question.question_text }}</title>
</head>
<body>
    <h1>{{ question.question_text }}</h1>
    
    {% if error_message %}<p><strong>{{ error_message }}</strong></p>{% endif %}
    
    <form action="{% url 'vote' question.id %}" method="post">
    {% csrf_token %}
    {% for choice in question.choice_set.all %}
        <input type="radio" name="choice" id="choice{{ forloop.counter }}" value="{{ choice.id }}">
        <label for="choice{{ forloop.counter }}">{{ choice.choice_text }}</label><br>
    {% endfor %}
    <input type="submit" value="Vote">
    </form>
</body>
</html>

2 Update Detail View

Update the detail view to use the template:

from django.shortcuts import get_object_or_404, render
from .models import Question

def detail(request, question_id):
    question = get_object_or_404(Question, pk=question_id)
    return render(request, 'polls/detail.html', {'question': question})

3 Implement Voting Logic

Update the vote view to handle form submissions:

from django.http import HttpResponse, HttpResponseRedirect
from django.shortcuts import get_object_or_404, render
from django.urls import reverse
from .models import Choice, Question

def vote(request, question_id):
    question = get_object_or_404(Question, pk=question_id)
    try:
        selected_choice = question.choice_set.get(pk=request.POST['choice'])
    except (KeyError, Choice.DoesNotExist):
        # Redisplay the question voting form
        return render(request, 'polls/detail.html', {
            'question': question,
            'error_message': "You didn't select a choice.",
        })
    else:
        selected_choice.votes += 1
        selected_choice.save()
        # Always return an HttpResponseRedirect after successfully dealing
        # with POST data. This prevents data from being posted twice if a
        # user hits the Back button.
        return HttpResponseRedirect(reverse('results', args=(question.id,)))

Testing Your Application

Django makes it easy to write tests for your application. Let's create some basic tests.

1 Create Tests

Open polls/tests.py and add the following tests:

import datetime
from django.test import TestCase
from django.utils import timezone
from .models import Question

class QuestionModelTests(TestCase):
    def test_was_published_recently_with_future_question(self):
        """
        was_published_recently() returns False for questions whose pub_date
        is in the future.
        """
        time = timezone.now() + datetime.timedelta(days=30)
        future_question = Question(pub_date=time)
        self.assertIs(future_question.was_published_recently(), False)

2 Run Tests

Run your tests with the following command:

python manage.py test polls
