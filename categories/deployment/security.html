Security Best Practices

20 min read Updated Jan 2024 Security
Important: Security is not optional. Follow these practices to protect your application and users.

Introduction

Django provides strong security protections out of the box, but proper configuration and development practices are essential for maintaining a secure application. This guide covers the most important security considerations for Django applications.

Note: Security is a process, not a one-time setup. Regularly review and update your security practices.

Django's Built-in Security Features

Django includes numerous security features that work automatically when properly configured:

Feature Description Status
Cross-Site Scripting (XSS) Protection Automatic escaping in templates Enabled by default
Cross-Site Request Forgery (CSRF) Protection CSRF tokens for POST requests Enabled by default
SQL Injection Protection ORM prevents SQL injection Enabled by default
Clickjacking Protection X-Frame-Options middleware Enabled by default
SSL/HTTPS Redirect Security middleware for HTTPS Requires configuration
Secure Headers Security middleware for security headers Requires configuration

Environment Configuration

Proper environment configuration is the foundation of Django security:

Secret Key Management

# settings.py
import os
from django.core.exceptions import ImproperlyConfigured

def get_secret_value(name, default=None):
    """Get secret value from environment variable."""
    value = os.getenv(name, default)
    if value is None:
        raise ImproperlyConfigured(f"Set the {name} environment variable")
    return value

SECRET_KEY = get_secret_value('DJANGO_SECRET_KEY')

Environment-based Settings

# settings.py
import os

DEBUG = get_secret_value('DJANGO_DEBUG', 'False').lower() == 'true'

ALLOWED_HOSTS = get_secret_value('DJANGO_ALLOWED_HOSTS', '').split(',')

# Database
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': get_secret_value('DB_NAME'),
        'USER': get_secret_value('DB_USER'),
        'PASSWORD': get_secret_value('DB_PASSWORD'),
        'HOST': get_secret_value('DB_HOST', 'localhost'),
        'PORT': get_secret_value('DB_PORT', '5432'),
    }
}
Warning: Never commit secret keys or passwords to version control. Use environment variables or secret management services.

Input Validation and Data Handling

Always validate and sanitize user input:

Form Validation

# forms.py
from django import forms
from django.core.exceptions import ValidationError
import re

class UserRegistrationForm(forms.Form):
    username = forms.CharField(max_length=150)
    email = forms.EmailField()
    password = forms.CharField(widget=forms.PasswordInput)
    
    def clean_username(self):
        username = self.cleaned_data['username']
        if not re.match(r'^[\w.@+-]+\Z', username):
            raise ValidationError("Enter a valid username.")
        return username

Model Validation

# models.py
from django.db import models
from django.core.exceptions import ValidationError

def validate_positive(value):
    if value < 0:
        raise ValidationError('Value must be positive')

class Product(models.Model):
    name = models.CharField(max_length=100)
    price = models.DecimalField(
        max_digits=10, 
        decimal_places=2,
        validators=[validate_positive]
    )
    
    def clean(self):
        if self.price == 0:
            raise ValidationError('Price cannot be zero')

File Upload Security

Secure handling of file uploads is critical:

# forms.py
from django import forms
from django.core.exceptions import ValidationError
import os

def validate_file_extension(value):
    ext = os.path.splitext(value.name)[1]
    valid_extensions = ['.pdf', '.doc', '.docx', '.jpg', '.png', '.txt']
    if not ext.lower() in valid_extensions:
        raise ValidationError('Unsupported file extension.')

def validate_file_size(value):
    filesize = value.size
    if filesize > 5 * 1024 * 1024:  # 5MB
        raise ValidationError("Maximum file size is 5MB")

class DocumentForm(forms.Form):
    document = forms.FileField(
        validators=[validate_file_extension, validate_file_size]
    )

SQL Injection Prevention

Django's ORM protects against SQL injection, but be careful with raw SQL:

Danger: Never use string formatting or concatenation with user input in SQL queries.

Safe Query Examples

# SAFE: Using Django ORM
users = User.objects.filter(username=request.GET['username'])

# SAFE: Parameterized raw SQL
from django.db import connection
with connection.cursor() as cursor:
    cursor.execute("SELECT * FROM users WHERE username = %s", [username])
    row = cursor.fetchone()

# UNSAFE: String formatting (VULNERABLE TO SQL INJECTION)
cursor.execute(f"SELECT * FROM users WHERE username = '{username}'")

XSS (Cross-Site Scripting) Prevention

Django templates automatically escape HTML, but be careful with mark_safe:

# SAFE: Automatic escaping in templates
<p>{{ user_input }}</p>  {# HTML is automatically escaped #}

# DANGEROUS: Using mark_safe incorrectly
from django.utils.safestring import mark_safe

# UNSAFE if user_input contains malicious JavaScript
content = mark_safe(user_input)

# SAFE: Only use mark_safe with trusted content
content = mark_safe("
Trusted content
")

Content Security Policy (CSP)

# Using django-csp package
# settings.py
MIDDLEWARE = [
    # ...
    'csp.middleware.CSPMiddleware',
]

CSP_DEFAULT_SRC = ["'self'"]
CSP_SCRIPT_SRC = ["'self'", "https://trusted-cdn.com"]
CSP_STYLE_SRC = ["'self'", "'unsafe-inline'"]

CSRF (Cross-Site Request Forgery) Protection

Django provides CSRF protection out of the box:

<!-- In templates -->
<form method="post">
    {% csrf_token %}
    <!-- form fields -->
    <button type="submit">Submit</button>
</form>
# For AJAX requests
const csrftoken = document.querySelector('[name=csrfmiddlewaretoken]').value;

fetch('/api/endpoint/', {
    method: 'POST',
    headers: {
        'X-CSRFToken': csrftoken,
        'Content-Type': 'application/json',
    },
    body: JSON.stringify(data)
})

Implement comprehensive logging and monitoring:

Security Monitoring

# middleware.py
import time
from django.core.cache import cache
from django.http import HttpResponseForbidden

class SecurityMiddleware:
    def __init__(self, get_response):
        self.get_response = get_response
        
    def __call__(self, request):
        # Rate limiting
        ip = self.get_client_ip(request)
        key = f"rate_limit:{ip}"
        requests = cache.get(key, 0)
        
        if requests > 100:  # 100 requests per minute
            return HttpResponseForbidden("Rate limit exceeded")
            
        cache.set(key, requests + 1, 60)
        return self.get_response(request)
        
    def get_client_ip(self, request):
        x_forwarded_for = request.META.get('HTTP_X_FORWARDED_FOR')
        if x_forwarded_for:
            ip = x_forwarded_for.split(',')[0]
        else:
            ip = request.META.get('REMOTE_ADDR')
        return ip

Security Testing

Regularly test your application for security vulnerabilities:

# Install security testing tools
pip install bandit safety django-debug-toolbar

# Run security linter
bandit -r myapp/

# Check for vulnerabilities in dependencies
safety check

# Use Django's check command
python manage.py check --deploy

Automated Security Testing

# tests/test_security.py
from django.test import TestCase
from django.urls import reverse

class SecurityTests(TestCase):
    def test_debug_mode_disabled(self):
        self.assertFalse(settings.DEBUG)
    
    def test_https_redirect(self):
        response = self.client.get('/', secure=False)
        self.assertEqual(response.status_code, 301)
        self.assertTrue(response.url.startswith('https://'))
    
    def test_csrf_protection(self):
        response = self.client.post(reverse('login'), {})
        self.assertEqual(response.status_code, 403)
